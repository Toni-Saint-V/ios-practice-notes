# Дизайн система расползлась из-за одного цвета

> **Коротко:** Один прямой `Color.blue` в feature-коде почти всегда размножается.

## Ситуация
В оплате сделали кнопку чуть ярче, чем системная primary. Потом такую же кнопку попросили в профиле. Потом в бронировании. Через месяц в проекте уже три «главные» кнопки, и каждая немного другая.

Дизайнер видит расхождение. QA видит расхождение. Пользователь не формулирует, но интерфейс кажется менее цельным.

## Что насторожило
- Feature-код сам выбирает цвет.
- Нет роли `primaryAction`.
- Loading/disabled state сделаны в каждом экране отдельно.
- Snapshot проверяет один экран, но не компонент.

## Лучше так
```swift
enum ActionKind {
    case primary
    case secondary
    case destructive
}

struct AppButton: View {
    let title: String
    let kind: ActionKind
    let isLoading: Bool
    let action: () -> Void

    var body: some View {
        Button(action: action) {
            Text(isLoading ? "Подождите" : title)
                .frame(maxWidth: .infinity, minHeight: 52)
        }
        .buttonStyle(AppButtonStyle(kind: kind))
        .disabled(isLoading)
    }
}
```

## Практический вывод
Дизайн-система должна запрещать легкие ошибки. Если проще написать `Color.blue`, чем взять токен, система проиграет.
